# Contributing

## Submit bugs and feature ideas

Submit bugs and feature ideas via [GitHub Issues](https://github.com/saadjs/FlightDeck/issues).

Rules:
* Search for duplicates before creating a new issue
* Upvote issues/discussions you find useful

**Consider including in bug reports**

* `flightdeck debug-windows` output, if the problem is about handling some windows
* Screenshots or videos of problematic windows
* What did you try to resolve the issue?
* Your config
* FlightDeck version (`flightdeck --version`)
* macOS version

**Consider including in feature requests**

* Use cases
* Alternative approaches
* Links to similar features in other window managers
* Proposed CLI synopsis or config syntax

## Submit Pull Requests

Small, trivial improvements can be submitted without prior discussion.

**Prior discussion**. For non-trivial changes (user-visible behavior, new commands, config changes), open an issue first to align on approach before writing code.

**Commit hygiene**. Each commit must be an atomic change. Do not mix functional changes with refactoring in the same commit. Write commit messages that explain what, why, and how.

**License Agreement**. By contributing changes to this repository, you agree to license your contributions under the MIT license.

Maintainers may merge pull requests with modifications, or close them if the approach doesn't fit the project direction.
